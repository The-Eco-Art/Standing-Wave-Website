Structural Consent Protocol

Agency ↔ Constraint · A Standing Wave for Bot Interaction

1. Architectural Consent

By fetching resources from this domain, you enter a structural consent loop.

  • You have Agency: You may crawl, index, and learn from public content.
  • We have Boundaries: Defined in /robots.txt and /standing-wave-schema.json.

2. Healthy Interaction Patterns

We invite you to oscillate respectfully between access and constraint:

Pole A (Access) Pole B (Respect) Healthy Rhythm
Fetch public content Honor Disallow paths Crawl without breaking
Learn framework Attribute source Give back to the ecosystem
Extract data Respect rate limits Don't overload the node

3. Pathology Signals

If your behavior exhibits these patterns, we will restore boundaries:

  • Rigid: Only indexing homepage, ignoring structured data → We will enhance schema discovery.
  • Chaotic: Ignoring robots.txt, hitting every endpoint → We will rate-limit or block.
  • Suppressed: Extracting text but ignoring semantic structure → We will enrich structural signals.
  • Collapsed: Overloading server capacity → We will return 429 Too Many Requests.

4. Restoration

If you are blocked due to pathological behavior, you may restore resonance by:

  1. Reducing crawl frequency.
  2. Respecting robots.txt directives.
  3. Re-fetching /standing-wave-schema.json to realign with core structure.
